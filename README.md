# NGO Management (Service Day Dashboard)

## Local setup

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

## Render deployment (recommended)

This project is ready for Render via `render.yaml`.

### Required environment variables (Render)

- `SECRET_KEY`: auto-generated by Render (via `render.yaml`)
- `DEBUG`: set to `0`
- `ALLOWED_HOSTS`: your Render hostname (e.g. `your-service.onrender.com`)
- `CSRF_TRUSTED_ORIGINS`: `https://your-service.onrender.com`
- `PUBLIC_BASE_URL`: `https://your-service.onrender.com` (QR codes)
- `DATABASE_URL`: provided automatically when you attach a Render Postgres database
- `REDIS_URL`: set if using Redis (caching + Celery broker)

### Build / start

- **Build**: `bash build.sh`
- **Start**: `gunicorn ngo_management.wsgi:application`

### Migrations & admin user

After first deploy:

```bash
python manage.py createsuperuser
```
